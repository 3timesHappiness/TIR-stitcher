# tir-stitcher 简要说明

> 面向无人机图像处理初学者 — 零基础可读

---

## 你的数据是什么？

DJI 无人机（如 Mavic 3T）拍的热红外照片（`T.JPG`）和手机照片不同：

| 普通照片 | 热红外照片 (T.JPG) |
|---------|-------------------|
| 看到颜色和亮度 | 看到温度 |
| RGB 三个通道 | 单通道温度值 |
| 没有温度信息 | 每个像素是一个温度数值 |

双击任意 `T.JPG`，你会看到一张模糊的灰度图。不是拍糊了，是热红外分辨率只有 640×512 像素。

---

## "拼接"要解决什么问题？

无人机在天上拍了上百张照片，每张只覆盖一小块地面：

```
单张覆盖 ≈ 60m × 80m          拼接后覆盖 ≈ 200m × 280m
┌────┐                        ┌────────────────────┐
│ ██ │                        │████████████████████│
└────┘                        │████████████████████│
                              │████████████████████│
                              └────────────────────┘
```

拼接就是把几百张小图拼成一张完整的大图。

---

## 工作流：6 步

```
你的 T.JPG 照片
    ↓  ① 提取温度数据 (DJI 专用工具)
  温度数值文件 (.raw)
    ↓  ② 转成标准图像格式
  TIF 图像 (无GPS)
    ↓  ③ 把 GPS 坐标写进图像
  TIF 图像 (有GPS，知道自己在哪)
    ↓  ④ 放到 ODM 认识的目录
  images/ 文件夹
    ↓  ⑤ ODM 自动配准 + 拼接
  正射影像 (大图)
    ↓  ⑥ 提取温度波段输出
  最终结果: 一张带有经纬度的温度地图
```

---

## 你只需要管 3 样东西

| 文件 | 作用 |
|------|------|
| `config.yaml` | 告诉程序：数据在哪？飞行高度多少？用 CPU 还是 GPU？ |
| `examples/` | 30 张示例照片，可以直接跑 |
| 启动命令 | `python -m tir_stitcher -c config.yaml` |

其余 18 个 Python 文件你不用管，它们自动工作。

## 动手实践（20 分钟）

### 1. 装依赖（2 分钟）

```bash
cd tir-stitcher
pip install -r requirements.txt
```

### 2. 看一眼配置（3 分钟）

用记事本打开 `config.yaml`，改成你的数据路径：

```yaml
workspace_dir: "你的项目文件夹路径"
```

其他参数保持默认即可，每个参数上面都有中文注释。

### 3. 先假装跑一遍（1 分钟）

```bash
python -m tir_stitcher -c config.yaml --dry-run
```

什么都不会真的做，只是打印它会做什么。可以确认：
- 找到了几个项目？
- 有没有工具缺失？（比如 Docker 没装）

### 4. 正式运行（10-15 分钟）

```bash
python -m tir_stitcher -c config.yaml -v
```

`-v` 会打印详细进度：

```
[1/6] extract_raw: 项目名
      从 T.JPG 提取 RAW 温度数据
      Processed 30/30 files

[2/6] raw_to_tif: 项目名
      将 RAW 转为 TIFF
      ...

... 以此类推
```

### 5. 看结果

运行结束后，项目文件夹里会多出：

```
你的项目/
├── RAW/                          ← 步骤① 产物
├── TIF/                          ← 步骤②③ 产物
├── images/                       ← 步骤④ 产物
├── orthophoto/
│   └── odm_orthophoto.tif        ← 步骤⑤ 产物（拼接好的大图！）
└── 项目名_TIR.tif                ← 步骤⑥ 最终输出
```

用 QGIS 打开 `_TIR.tif` 就能看到拼接完成的温度地图。

## 需要装什么工具？

| 工具 | 用途 | 下载 |
|------|------|------|
| **Python 3.10+** | 运行脚本 | [python.org](https://www.python.org/) |
| **DJI Thermal SDK** | 从 T.JPG 提取温度 | [DJI 下载中心](https://www.dji.com/global/downloads/softwares/dji-thermal-sdk) |
| **ExifTool** | GPS 坐标写入 | [exiftool.org](https://exiftool.org/) |
| **Docker** | 图像拼接（第⑤步） | [docker.com](https://www.docker.com/) |

如果暂时没有 Docker，前 4 步仍然可以跑。Docker 装好后，由于断点续传，前 4 步会自动跳过。

## ODM 是什么？

[OpenDroneMap](https://www.opendronemap.org/)（简称 ODM）是流水线第⑤步使用的开源航拍图像拼接引擎。它通过 Docker 容器运行，自动将几百张带 GPS 坐标的图像：
- 提取特征点 → 匹配相邻照片
- 运动恢复结构（SfM）→ 重建三维场景
- 生成正射影像（俯瞰大图）

**相关链接：**
- 官网：[opendronemap.org](https://www.opendronemap.org/)
- 文档：[docs.opendronemap.org](https://docs.opendronemap.org/)
- GitHub：[OpenDroneMap/ODM](https://github.com/OpenDroneMap/ODM)
- Docker Hub：[opendronemap/odm](https://hub.docker.com/r/opendronemap/odm)

**本流水线中的 ODM 关键参数（已自动配置，无需手动改）：**

```
--radiometric-calibration none   # 保留原始温度值，不做 RGB 转换
--primary-band 1                 # 只处理温度波段
--feature-type sift              # SIFT 算法对热红外纹理最友好
--orthophoto-png                 # PNG 无损输出
```

## 断点续传

每个阶段完成后会在项目文件夹里生成一个 `.tir_stitcher_xxx.done` 标记文件。下次运行时，已完成的阶段会自动跳过。

**重跑某个阶段**：删除对应的 `.done` 文件，然后重新运行。

| 重跑... | 删除... |
|---------|---------|
| 提取 RAW | `.tir_stitcher_extract_raw.done` |
| RAW → TIF | `.tir_stitcher_raw_to_tif.done` |
| 复制 EXIF | `.tir_stitcher_copy_exif.done` |
| 准备 images | `.tir_stitcher_prepare_images.done` |
| 运行 ODM | `.tir_stitcher_run_odm.done` |
| 后处理 | `.tir_stitcher_postprocess.done` |

## 遇到问题？

| 现象 | 排查方向 |
|------|----------|
| `dji_irp.exe not found` | 在 `config.yaml` 的 `tsdk.exe_path` 里填 SDK 路径 |
| `exiftool not found` | 在 `config.yaml` 的 `copy_exif.exiftool_path` 里填 exiftool 路径 |
| `Docker not found` | 安装 Docker Desktop，确保右下角图标是绿色的 |
| `Need at least 3 images` | ODM 至少需要 3 张重叠的照片才能拼接 |
| RAW 尺寸不对 | 在 `config.yaml` 里手动设置 `raw_to_tif.rows` 和 `cols` |
| 拼接失败 / 黑屏 | 降低 `odm.min_num_features` 到 4000，或用 `feature_quality: high` |
